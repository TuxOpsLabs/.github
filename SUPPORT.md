# Support

## Getting Help

The best place to start depends on the type of question.

### Bug Reports

Please use GitHub Issues when available for the repository.

Include:

* Steps to reproduce
* Expected behavior
* Actual behavior
* Relevant logs or screenshots

### Feature Requests

Feature requests and suggestions are welcome through GitHub Issues or Discussions when enabled.

### Security Issues

Please do not report security vulnerabilities publicly.

Refer to the Security Policy for responsible disclosure procedures.

## Project Status

Many TuxOps projects are actively evolving.

Documentation, APIs and features may change as projects mature.

## Response Times

As an independent technology studio, support is provided on a best-effort basis.

While we try to respond promptly, response times may vary.

Bear in mind, we're not a company with a dedicated support team, but rather a small team of passionate developers.

## Community Contributions

Community feedback, bug reports and contributions are greatly appreciated and help improve the quality of our projects.

Thank you for your interest in TuxOps.
