# Contributing

First of all, thank you for your interest in contributing.

While many TuxOps repositories are currently developed internally, we strongly believe in collaboration and knowledge sharing.

## Before Contributing

Please:

1. Review existing issues and discussions.
2. Open an issue before starting large changes.
3. Keep pull requests focused and easy to review.

## Development Principles

We value:

* Simplicity
* Maintainability
* Readability
* Long-term thinking
* Consistent architecture

We generally prefer solutions that are easy to understand and maintain over unnecessarily complex implementations.

## Pull Requests

When submitting a pull request:

* Clearly explain the purpose of the change.
* Keep changes focused on a single concern.
* Update documentation when appropriate.
* Follow the conventions used within the repository.

## Code Style

Each repository may define additional coding standards and development workflows.

Repository-specific documentation always takes precedence over this document.

## Communication

Please be respectful and constructive when participating in discussions.

We welcome feedback, suggestions and thoughtful debate.

## Licensing

By contributing, you agree that your contributions may become part of the repository under its applicable licensing terms.
